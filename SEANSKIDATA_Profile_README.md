## Featured Project

### [WFM Planning & Optimization Engine](https://github.com/SEANSKIDATA/WFM-Demand-Forecasting)

An end-to-end workforce management planning workflow built on real contact-center arrival data (Technion "Anonymous Bank" dataset, 87,600 rows).

**What it demonstrates:**

| Module | Description | Status |
|--------|-------------|--------|
| Forecasting | Interval-level volume forecast, holdout-validated (WAPE 21.2%) | ✅ Complete |
| Capacity Planning | Erlang C staffing, shrinkage build-up, FTE sizing | ✅ Complete |
| Gap Analysis | Required vs scheduled, interval-level, delivered SL | ✅ Complete |
| Optimization Engine | Greedy redistribution — +20.2% SL recovery, zero new hires | ✅ Complete |
| Decision Lab | What-if simulator: redistribution, AHT, shrinkage, headcount | ✅ Complete |
| Multichannel | Voice + Chat (Erlang C + concurrency) + Email (backlog model) | ✅ Complete |

**Key finding:** Total scheduled headcount within ~1.6% of requirements — yet 53% of intervals understaffed, 44% overstaffed, delivered service level 62.4% vs 80% target. The optimization engine recovers 20.2% service level through redistribution alone.

> *The problem wasn't the total headcount. The problem was when people were scheduled.*
